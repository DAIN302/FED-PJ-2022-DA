[ DC PJ 분석방향 ]

- 대상사이트 : http://dc.com

1. 분석의 기준
   (1) 기능별로 구성이 독립된 모듈로 분석함
   예) 배너스타일, 스와이프메뉴스타일, 요약소개스타일...
   모듈화하여 별도의 명칭으로 로딩

   (2) 기능별 구성의 데이터구조를 일치시키기 위한 분석
   예) 배너스타일이 다른 카테고리별로 내용만 다름
   -> 데이터 구조가 같음, 분류체계는 같고 내용만 다름

2. 카테고리 구성
   (1) 내용별 구성보다는 유형별 구성을 할 것
   -> 아무개소개 X =>>> 요약소개모듈
   (2) 카테고리 구성전개를 유형구성으로 작성
   예) 상단동일모듈+배너모듈+요약소개모듈1+요약소개모듈2+
   스와이퍼메뉴모듈+동영상모듈+단순링크모듈+하단동일모듈
   (3) 카테고리별 모듈구성으로 작성후 그 모듈에 대한 카테고리 데이터를 매칭
   -> 같은 유형의 모듈에 실제로 들어올 데이터 구조에 맞는 실제데이터를 표시
   예) 비디오 카테고리 상단에 "동영상모듈"이 들어옴
   -> 데이터구조 : 비디오 + 제목 + 설명
   비디오정보 + 제목내용 + 설명내용
   {vinfo : "", tit :"", desc : ""}
   -> 데이터구조 즉, 객체화구조를 뽑아냄
   -> 실제 들어가는 데이터는 표시하지 않아도 됨(더미데이터로 표시)
